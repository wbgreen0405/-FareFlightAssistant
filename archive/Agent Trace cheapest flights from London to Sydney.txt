To find the cheapest flights from London to Sydney, leaving on the 16th of December 2023 and returning after the 19th of January 2024, I'll write an SQL query that:

1. Searches for flights that have segments departing from London (LHR) on the 16th of December 2023 and arriving in Sydney (SYD).
2. Searches for the return flights that have segments departing from Sydney (SYD) after the 19th of January 2024 and arriving in London (LHR).
3. Ensures that the segments are part of a valid itinerary linked to a flight offer.
4. Aggregates the total price for each valid round-trip journey and selects the one with the lowest total price.

Here is the SQL query that accomplishes this:

```sql
WITH DepartureFlights AS (
    SELECT 
        fo.FlightOfferID,
        fo.TotalPrice AS OutboundPrice,
        i.ItineraryID,
        s.DepartureTime AS DepartureDateTime
    FROM 
        FlightOffer fo
        JOIN Itinerary i ON fo.FlightOfferID = i.FlightOfferID
        JOIN Segment s ON i.ItineraryID = s.ItineraryID
    WHERE 
        s.DepartureIATACode = 'LHR' AND
        s.ArrivalIATACode = 'SYD' AND
        DATE(s.DepartureTime) = '2023-12-16'
),
ReturnFlights AS (
    SELECT 
        fo.FlightOfferID,
        fo.TotalPrice AS ReturnPrice,
        i.ItineraryID,
        s.DepartureTime AS ReturnDateTime
    FROM 
        FlightOffer fo
        JOIN Itinerary i ON fo.FlightOfferID = i.FlightOfferID
        JOIN Segment s ON i.ItineraryID = s.ItineraryID
    WHERE 
        s.DepartureIATACode = 'SYD' AND
        s.ArrivalIATACode = 'LHR' AND
        DATE(s.DepartureTime) > '2024-01-19'
),
CombinedFlights AS (
    SELECT 
        df.FlightOfferID AS OutboundFlightOfferID,
        rf.FlightOfferID AS ReturnFlightOfferID,
        df.OutboundPrice,
        rf.ReturnPrice,
        (df.OutboundPrice + rf.ReturnPrice) AS TotalRoundTripPrice
    FROM 
        DepartureFlights df
        CROSS JOIN ReturnFlights rf
)
SELECT 
    cf.OutboundFlightOfferID,
    cf.ReturnFlightOfferID,
    cf.OutboundPrice,
    cf.ReturnPrice,
    cf.TotalRoundTripPrice
FROM 
    CombinedFlights cf
ORDER BY 
    cf.TotalRoundTripPrice ASC
LIMIT 1;
```

This query returns the FlightOfferID for the outbound and return flights, along with their individual prices and the total price for the round trip. We use a CROSS JOIN because each outbound flight can be combined with any return flight, and we're interested in finding the combination with the lowest total price.

Keep in mind that this query assumes that there is a direct correlation between FlightOfferID and the complete journey from London to Sydney and back. In a real-world scenario, we might need to accommodate for multi-leg flights and ensure that the itineraries connect properly. However, with the provided schema, this is the best approach to get the desired results.
